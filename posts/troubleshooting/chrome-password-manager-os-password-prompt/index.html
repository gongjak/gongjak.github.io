크롬 비밀번호 자동완성 시 macOS 비밀번호 입력창이 뜨는 문제 해결

March 31, 2026 · 1 분
목차
  • 증상
  • 원인
  • 해결
    • 1. Google 비밀번호 관리자 설정 열기
    • 2. 화면 잠금 사용 끄기
  • 검색 팁
  • 참고

증상#

크롬에서 웹사이트에 로그인할 때 저장된 비밀번호를 자동완성하려고 하면, 아래처럼 macOS 계정 비밀번호를 입력하라는 팝업이 매번 나타난다.

“Google Chrome.app"이(가) 키체인의 비밀번호를 사용하려고 합니다.

매번 비밀번호를 입력하거나 Touch ID를 사용해야 하니 상당히 번거롭다. 키체인 설정이 바뀐 줄 알고 한참 헤맸는데, 실제 원인은 크롬의 Google 비밀번호 관리자 설정이었다.

원인#

Chrome의 Google 비밀번호 관리자에 “비밀번호 입력 시 화면 잠금 사용” 옵션이 활성화되어 있으면, 저장된 비밀번호를 자동완성할 때마다 OS 수준의 인증(macOS 비밀번호 또는 Touch ID)을 요구한다.

보안 관점에서는 의미 있는 기능이지만, 개인 PC에서는 과도한 보안이 되어 사용성을 해친다.

해결#

1. Google 비밀번호 관리자 설정 열기#

크롬 주소창에 아래 URL을 직접 입력하면 빠르게 이동할 수 있다.

chrome://password-manager/settings

또는 수동으로: 크롬 설정 → 자동 완성 및 비밀번호 → Google 비밀번호 관리자 → 설정

2. 화면 잠금 사용 끄기#

“비밀번호 입력 시 화면 잠금 사용” 토글을 OFF로 변경한다.

이 설정을 끄면 비밀번호 자동완성 시 더 이상 macOS 비밀번호를 묻지 않는다.

검색 팁#

이 문제를 검색할 때 “크롬 키체인 비밀번호"로 찾으면 관련 없는 결과가 많이 나온다. 영문으로 **“google password manager asking for password”**로 검색하면 정확한 해결책을 빠르게 찾을 수 있다.
